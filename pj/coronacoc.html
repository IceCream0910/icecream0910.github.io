Summary

코로나19라는 새로운 바이러스의 등장은 우리의 평범했던 일상을 앗아갔습니다.
코로나콕은 다양한 통계들과 정보를 한눈에 보기 쉽게 시각화하여 보여주는 서비스입니다.

보건당국에서 제공하는 통계 수치들을 그래프로 시각화하여 제공합니다.

어려웠던 점 & 해결방안

혼자서 개발하다 보니 발생하는 오류들에 곧바로 대응하기에도 어려움이 있었다.
최대한 빠르게 대응하기 위해 채널톡을 활용해 사용자들이 채팅으로 오류나 문의사항들을 알려줄 수 있도록 했다.
코로나19 관련 서비스 지원을 받아 무료로 유료 플랜을 사용해 더욱 효과적으로 응대가 가능했다.

newsapi.org의 api url을 직접 호출했을 때 403오류가 발생했다.
구글링을 해보니 newsapi.org의 정책 변경으로 무료 버전에서는 api를 localhost에서만 사용할 수 있게 되었다고 한다. 이를 해결하기 위해서 node.js로 백엔드 api 서버를 만든 후 heroku를 활용해 호스팅해 해결했다.

서비스를 로딩하는데 너무 오랜 시간이 소요되는 경우가 발생했다.
html에서 style 태그를 직접 정의한 경우가 많았고, 탭 기능을 구현할 때 iframe을 복수 사용하다 보니 document로딩 시간이 길어질 수 밖에 없었다.
이러한 문제를 해결하기 위해 디자인 개편을 진행함과 동시에 코드 리팩토링을 진행해 성능을 개선하고자 하였다. 또 js와 css를 경량화(minify)하는 방식을 이용해 성능을 극대화할 수 있었다. 또한 디자인 개편 과정에서 애니메이션 효과를 이용한 로딩 화면을 추가해 사용자가 체감하는 로딩 속도를 개선할 수 있었다.

안드로이드 푸시 알림을 구현할 때 재부팅 시 푸시 알림이 발송되는 이슈가 있었다.
매일 오전 10시 30분에 신규 확진자를 서버에서 불러온 후 푸시 알림을 발송해주는 기능을 구현하였고, 인터넷이 연결되지 않았을 때는 '인터넷이 연결되지 않아 현황을 불러오지 못했다'라는 푸시 알림을 발송하도록 했다.
그러나 재부팅을 하고 난 후에 10시 30분이 아닌데도 곧바로 인터넷에 연결되어 있지 않다는 메시지가 송출되었다. manifest 파일에서 BOOT_COMPLETED intent를 알림을 송출하는 리시버가 아닌 아닌 알림 예약 리시버에만 추가해주는 방법으로 해결했다.

새롭게 알게된 점

서비스를 배포하고 운영해보는 경험
내가 만든 서비스를 공개적으로 배포하였고, 코로나19에 대한 관심에 힘입어 예상보다는 많은 사용자를 확보할 수 있었다(물론 제 기준에서). 서버부터 호스팅까지 모두 비용을 지불하지 않고 개발을 진행하려다 보니 금전적인 어려움에 부딪히기도 했지만 최대한 비용을 지불하지 않고 트래픽을 낮추거나 지원을 받는 방향으로 새로운 경험을 해볼 수 있었다.

사용자의 피드백
서비스를 사용하는 사용자들이 내가 미처 생각치 못한 오류나 아이디어들을 제보/제안해 줄 때가 많았다. 사용자가 불편을 겪을 만한 오류들을 빠르게 해결해야 한다는 책임감을 가지게 되었으며, 사용자가 원하는 기능을 어떻게든 서비스에 녹여내보려 하며 그 속에서 새로운 기술들을 배울 수 있었다.

Notion 사용
비록 1인 개발이었지만, 사용자의 피드백을 바탕으로 해결해야 할 오류나 개선점, 떠오르는 아이디어들을 notion이라는 툴을 이용해 정리하였다.
무엇을 먼저 진행해야 할지 우선순위를 정해 보다 효율적으로 서비스를 유지보수할 수 있었다.

새로운 기술
이 서비스를 개발하며 많은 기술을 새로 접하게 되었다. 서비스에 직접 사용하지는 않았지만 Vue, React와 같은 기본적인 프레임워크들도 알게 되었고, 게시판 기능을 구현하며 Firebase라는 툴도 사용해보게 되었다.
웹페이지를 이용해 안드로이드 하이브리드 앱을 만들어보는 경험을 통해 웹과 앱 간에 통신을 가능하게 해주는 BRIDGE 등을 이용해 기존에 했던 네이티브 앱과는 또다른 방식의 앱을 개발해보는 경험을 하게 되었다.
